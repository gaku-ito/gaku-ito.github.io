1 RによるAPIの活用

API (Application Program Interface) は，ソフトウェア間で機能を共有する仕組み (命令，関数等の集合) のことをいう．APIを用いることで，遠隔地にあるコンピュータやサーバーが提供する機能・データを，他のソフトウェア (ここではR) で利用できる．

近年の大規模なデータベースでは，「クリック」してデータセット (の一部) をダウンロードするサービスに加えて，APIを用いてデータセットを取得できることが多い． 昨日の授業で利用したUN ComtradeでもAPIが利用可能で，これを利用すれば「ポチポチ」ではなくRを通してデータを取得し解析に進むことができる．

以下ではAPI一般の技術的な解説は省略し，UN ComtradeのAPIを利用してデータを取得するRコードの解説に集中する．

3 UN Comtrade データセットの取得

素晴らしいことに，UN ComtradeではAPIを用いてデータセットを取得するRの関数を提供している (公式サイト)．ただし，このRコードは一部が既に利用できず，また関数にも改善点があるので修正を加えた新たな関数を定義しよう．

3.1 国コードの取得 (注意：修正あり)

3.2 Comtrade取得用関数

さて，準備が整ったところで，Comtradeのデータを取得するためのget.Comtrade.data()関数を定義する．

get.Comtrade.data <- function (
    reporter,           ## reporter country
    partner = "all",    ## partner country
    url.prefix = "http://comtrade.un.org/api/get?", maxrec = 50000,
    type = "C", ## commodity (C) or service (S)
    freq = "A", ## A=annually, M=monthly
    trade.code = "HS",  ## HS=HS code as reported, S1=SITC code rev.1
    period = "now",
    trade.flow = "all", ## trade flows
    commodity.code = "TOTAL",   ## commodity code
    return.url = FALSE, ## return url or not
    output.format = "csv",      ## output file-format: jason or csv
    agent = "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_11_4) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/50.0.2661.86 Safari/537.36"
    ) {

    require(RCurl)
    require(RJSONIO)
    require(tidyverse)
    
    ## check output format argument
    if (!output.format %in% c("csv", "json", "url")) {
        stop("Type must be either 'csv,' 'json', 'url'")
    }

    ## generate url
    string <- str_c(url.prefix, "max=", maxrec, "&", "type=", type,
        "&freq=", freq, "&", "px=", trade.code, "&", "ps=", period, "&",
        "r=", reporter, "&", "p=", partner, "&", "rg=", trade.flow, "&", "cc=", commodity.code,
        "&", "fmt=", output.format,
        "&head=M"           ## header as in json query
        )

    ## extract data
    curl = getCurlHandle()
    curlSetOpt(cookiejar="cookies.txt",  useragent=agent, followlocation=TRUE, curl=curl)
    firstPage <- getURL("http://comtrade.un.org", curl=curl)
    # ------------------------------------------------------------
    if (output.format == "csv") {
        if (return.url) {
            
            return(getURL(string, curl=curl))
        } else {
            tmp.data <- read_csv(getURL(string, curl=curl), col_types = cols(.default = "c"))
            return(list(validation = NULL, data = tmp.data))
        }
    } else {
        if (return.url) {
            return(getURL(string, curl=curl))
        } else {
            tmp.data <- RJSONIO::fromJSON(getURL(string, curl=curl))
            data <- tmp.data$dataset
            validation <- unlist(tmp.data$validation, recursive = TRUE)
            ndata <- NULL
            if (length(data) > 0) {
                var.names <- names(data[[1]])
                data <- as.data.frame(t(sapply(data, rbind)))
                ndata <- NULL
                for (i in 1:ncol(data)) {
                  data[sapply(data[, i], is.null), i] <- NA
                  ndata <- cbind(ndata, unlist(data[, i]))
                }
                ndata <- as.data.frame(ndata, stringsAsFactors=FALSE)
                colnames(ndata) <- var.names
                ndata <- ndata %>% 
                    tbl_df %>%
                    mutate_each(funs(type.convert(as.character(.))))
            }
            return(list(validation = validation, data = ndata))
        }
    }
    on.exit(close(string))
}

この関数をRコードとして完全に理解する必要はなく，背後にある仕組みと使い方を理解すればよい．Comtradeのウェブページに解説がある通り，Comtradeのデータは一定の一定のパラメータ (e.g., 年，国，フローの種類) を指定してやると，その条件でフィルタリングしたデータセットの一部を，jsonあるいはcsv形式で返してくれる．

get.Comtrade.data()関数はこの仕組み (API) を利用して，引数として与えられたパラメータを利用してComtradeのデータベースにアクセスする．その上で，(1) Comtradeのデータを (jsonあるいはcsv形式で) Rに直接読み込むか (return.url = FALSE; デフォルト)，(2) csvファイルダウンロード用のurlを生成する (return.url = TRUE)．return.url = FALSEの場合，get.Comtrade.data()関数はリストオブジェクトを返し，1つ目の要素にはデータ取得日時や指定されたデータを問題なく取得できたか等の情報が，2つ目の要素にはデータセットが格納される．return.url = TRUEとした場合は，ダウンロード用のurlを格納した文字列ベクトルを返す．return.url = TRUEとして取得したurlにブラウザでアクセスしても，指定したデータを取得することができる．

3.4 取得するデータをより細かく指定する

上記の例では，日本の国コードをreportersオブジェクトから取得した．このコードは貿易相手国をpartnerという引数で指定する際にも用いることができる．たとえば，カナダと中国の国コードを取得したければc("Japan")を変更すればよい．

jp_ccode = reporters$ccode[reporters$cname %in% c("Japan")] # 日本の国コードを取得する
cc_ccode = reporters$ccode[reporters$cname %in% c("Canada", "China")] # カナダ，中国の国コードを取得する

もし日本とカナダ・中国との2014年の貿易データのみを取得したければ，cc_ccodeを用いて

cc_ccode = str_c(cc_ccode, collapse=",")
jp_cc_sample = get.Comtrade.data(jp_ccode, partner = cc_ccode, output.format = "json", period = 2014)

とすれば，指定したデータを取得できる．なお，輸出・輸入や貿易コード，特定の品目コードも同様に指定できる． どのような値を取得できるかは関数に適宜コメントを入れているが，詳細はComtradeのウェブページ を参照すること．

4 注意点

このように，APIを用いれば「ポチポチ」して取得しなければならないデータを簡単に取得できる．この関数とComtradeは本講義の課題や卒論等に用いても構わないが，使用したい場合は次の点に十分注意すること．

  1. 本講義の担当教員に相談する
    • ComtradeのAPIやget.Comtrade.data()関数のクセを理解して使用しないと，誤ったデータを取得する可能性がある
    • クレジット云々というよりも，こうした誤ったデータ取得を防ぐため，事前に本講義の担当教員に相談すること
  2. 一定時間内のアクセスを「ある程度」に抑える
    • ComtradeへのアクセスやAPI利用は，有料アカウントを保持しているか否かによって制限される．
    • 詳細はComtradeのウェブページに案内がある が，有料アカウントでなければ1時間あたり100回に制限されている．100回/1hという制限は，コードを書けば容易に超えてしまうので注意する必要がある．
    • また，あまりにアクセスが多いと，最悪「こいつはComtradeに攻撃をかけている」と誤解されアクセスを遮断されるかもしれないので，(特に大学からアクセスするときには) 十分注意すること (こうした制限・問題のため，研究で使用する際には有料ライセンスを購入することが多い)
    • たとえばfor文を使って繰り返しアクセスする際には，1回データを取得する度にSys.sleep(1) 等を用いて時間を空けることが望ましい (Sys.sleep(1)は「1秒ポーズ」．Sys.sleep(0.5)なら0.5秒)
  3. 1度に取得するデータの行数を「ある程度」に抑える
    • 上記と同様に，1度に取得できるデータの規模にも制限がある．
    • 詳細はComtradeのウェブページに案内がある が，有料アカウントでなければ1度に取得できる最大のデータ行数は50000行に制限されている．
    • この行数は，貿易品目を細かく指定するとすぐに超えてしまうので，詳細な貿易品目別データを取得したい場合，なるべく「細切れ」にしてデータを取得する必要がある．同時に，上記のアクセス回数制限があるので，ある種のトレード・オフが存在する．
